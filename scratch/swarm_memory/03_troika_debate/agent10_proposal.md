# Agent 10: Proposal to Resolve Architectural Contradictions

## 1. Resolution to Methodological Contradiction (Dynamic Prompt Tuning vs. Scientific Replicability)
**Fix:** Replace dynamic runtime prompt tuning with a "version-controlled static protocol". Subagent extraction prompts must be generated *a priori* from `.agents/AGENTS.md` and the `03_Coding_Rulebook`, and frozen as a static template for an entire extraction batch. Rule 30 should be amended to restrict "dynamic" capabilities to the selection of pre-compiled, static protocol versions rather than altering prompt text mid-pipeline. Any necessary prompt changes require a formal rulebook update, version bump, and full re-extraction of the batch.

## 2. Resolution to Operational Contradiction (Strict Data Isolation vs. Rogue Code Execution)
**Fix:** Refactor `swarm_inject.py` and `swarm_prep.py` (and any related injection scripts) to strictly adhere to Rule 25. Legacy hardcoded paths (`BSMA_AI_Run_V2.xlsx`) must be updated to target `BSMA_Master_Coding_Sheet.xlsx`. The logic indiscriminately writing to `wb.active` must be replaced with precise sheet-routing logic. Scripts must inspect the data's flags (e.g., `is_transformed`, `is_imputed`) and route data exclusively to the correct physical sheet: `Raw_Metrics`, `Transformed_Metrics`, `Imputed_Metrics`, or `Salami_Review_Queue`.

## 3. Resolution to Epistemological Contradiction (Zero-Guesswork vs. Swarm Debate Consensus)
**Fix:** Restrict the 13-agent Swarm OS scope. Primary data extraction must be handled exclusively by non-debating, single-shot extraction subagents adhering to strict literal extraction rules. The Swarm OS debate mechanism should only be invoked for *validation* (e.g., confirming a literal match) or *triage* (e.g., evaluating inclusion criteria). Debate must be explicitly prohibited for resolving missing data; if a value is not explicitly stated in the text, the debate phase is bypassed, and the Rule 1 fallbacks (`999`, `"Not Reported"`) are immediately applied.
