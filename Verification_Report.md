# Include/Exclude Loop Pipeline Verification Report

## Overview
This report documents the verification of the Include/Exclude screening loop pipeline located in `05_Pipeline/1_Include_Exclude_Loop/` against the procedural methodology defined in `include_exclude_pipeline` (`SKILL.md`). The verification was conducted via static analysis and a complete end-to-end dry-run with a mock batch size of 2 on a copy of the Excel master sheet (`Test_Coding_Sheet.xlsx`).

## Step-by-Step Compliance Mapping

### Step 1: Batch Payload Generation
* **Skill Requirement**: Run `python 05_Pipeline/1_Include_Exclude_Loop/swarm_prep.py --batch 40` to scan the Excel file and generate a payload at `scratch/subagents_payload.json` containing up to 40 pending papers. If no more pending papers are found, proceed to Step 4 (Reporting).
* **Python Execution Logic**:
  * `swarm_prep.py` loads the spreadsheet defined by `EXCEL_PATH` and scans from row 4 downward.
  * A row is selected if the article ID (column 2) starts with "BSMA" and is not empty, and the status (column 5) is empty, `None`, or whitespace.
  * For each selected row, the script calls `find_pdf_for_id()` to locate the corresponding PDF in `01_Academic_Papers/` matching `[{num}]` (e.g., `BSMA0001` -> `[1]`).
  * The PDF file path is normalized using forward slashes for prompt compatibility.
  * The script writes a JSON payload array containing subagent configs (`TypeName: bsma_reviewer_v2`, `Role`, `Prompt`) to `scratch/subagents_payload.json`.
  * If no papers are pending, it prints `"No more pending papers found!"` and exits.
* **Findings & Fixes**:
  * **Critical Bug Found**: `swarm_prep.py` was hardcoded to read from `BSMA_AI_Run_V2.xlsx`, whereas `swarm_inject.py` was configured to write to `BSMA_Master_Coding_Sheet.xlsx`. This broke the loop, causing `swarm_prep.py` to continuously prepare the same pending batch from the un-updated `BSMA_AI_Run_V2.xlsx` (an infinite reprocessing loop).
  * **Permanent Fix Applied**: Updated `EXCEL_PATH` on line 7 of `swarm_prep.py` to `BSMA_Master_Coding_Sheet.xlsx`.
* **Compliance Verdict**: **PASS** (with permanent path fix applied).

### Step 3: Data Injection & Loop
* **Skill Requirement**: Once subagents finish, run `python 05_Pipeline/1_Include_Exclude_Loop/swarm_inject.py` to write their verdicts into the Excel sheet and clean up the JSON files. After injection, loop back to Step 1.
* **Python Execution Logic**:
  * **Gathering**: `swarm_inject.py` scans subagent brain directories (`C:\Users\yunky\.gemini\antigravity\brain\*\scratch\outputs\BSMA*.json`) and copy them to `scratch/outputs/`.
  * **Injection**: It parses each JSON, matches the article ID against Column 2 in the Excel sheet, and injects:
    * `status` into Column 5 (Status)
    * `reason_code` into Column 6 (Reason Code)
    * A concatenated note: `"{reason_summary}. Verbatim Evidence: \"{verbatim}\""` into Column 16 (Notes).
  * **Cleanup**: It deletes the processed JSON files to prevent double-processing on subsequent loops.
  * **Reporting**: It calls `generate_batch_report.py --ids <injected_ids>` to compile a Markdown batch report in `04_Reports/Batch_Report_<timestamp>.md`.
  * **Highlighting & Git Sync**: It runs `post_highlight.py` to scan Column 16 in the Excel sheet using Windows COM automation (`win32com.client`), styling matching keywords (`team`, `firm`, `students`, `alliance`) in red and bold. Finally, it executes `git add`, `git commit`, and `git push` to save changes to the remote repository.
